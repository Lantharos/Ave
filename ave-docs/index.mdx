Ave is an **OAuth 2.0 + OpenID Connect provider** built around passkey-based, passwordless identity. It handles sign-in, token issuance, delegated app access, cryptographic signing, and optional per-app encryption key delivery.

<Tip>
  **Just want to ship something?** Skip the setup entirely. [Quick Ave](/guides/quick-ave) gets you working auth in 3 lines with no registration, no client ID, and no environment variables.
</Tip>

## Key URLs

<Warning>
  Ave uses **two separate hosts**. The sign-in UI, OIDC discovery, and JWKS are on `aveid.net`. All API calls (token exchange, userinfo, signing) go to `api.aveid.net`.
</Warning>

<AccordionGroup>
  <Accordion title="Auth UI endpoints">
    | Purpose | URL |
    |---|---|
    | Sign-in page | `https://aveid.net/signin` |
    | Connector consent page | `https://aveid.net/connect` |
  </Accordion>
  <Accordion title="API endpoints">
    | Purpose | URL |
    |---|---|
    | OAuth token endpoint | `https://api.aveid.net/api/oauth/token` |
    | Userinfo endpoint | `https://api.aveid.net/api/oauth/userinfo` |
    | OIDC discovery | `https://aveid.net/.well-known/openid-configuration` |
    | JWKS (JWT verification keys) | `https://aveid.net/.well-known/jwks.json` |
  </Accordion>
</AccordionGroup>
