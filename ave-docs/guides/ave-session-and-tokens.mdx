The OAuth **`access_token` JWT** and **`id_token`** expire on a short clock (often about **one hour** by default). That is separate from how long a *user* should stay signed in: with **`offline_access`**, Ave issues a **`refresh_token`** so your app can rotate tokens without sending the user through the consent screen again.

The painful part in real apps is **session semantics**: duplicated refresh calls, lost refresh rotation, races on cold start, and Convex receiving an expired `id_token`. **`AveSession`** in `@ave-id/sdk` encodes one supported lifecycle: persist the bundle, refresh in a safety window before expiry, single in-flight refresh, and atomic save after rotation.

## When to use AveSession

- You want **`convex.setAuth`** to always receive a **fresh `id_token`**.
- You use a **registered** OAuth app and requested **`offline_access`** (Quick Ave does not issue refresh tokens â€” see [Upgrading from Quick Ave](/guides/quick-ave-to-standard)).

## Basic usage

```ts
import {
  AveSession,
  createLocalStorageAdapter,
  snapshotFromTokenResponse,
} from "@ave-id/sdk";
import { exchangeCode } from "@ave-id/sdk";

const session = new AveSession({
  oauth: {
    clientId: process.env.AVE_CLIENT_ID!,
    redirectUri: "https://yourapp.com/callback",
  },
  storage: createLocalStorageAdapter(),
});

await session.hydrate();

// After redirect callback â€” exchange code, then:
const tokens = await exchangeCode(
  { clientId: process.env.AVE_CLIENT_ID!, redirectUri: "https://yourapp.com/callback" },
  { code, codeVerifier }
);
await session.setTokensFromResponse(tokens);

const idToken = await session.getValidIdToken(); // for Convex
```

`getValidIdToken()` awaits proactive refresh when the JWT is near expiry. Prefer **`createMemoryStorage()`** or a **BFF** for production refresh-token handling instead of `localStorage` (see [Quickstart](/quickstart) storage guidance).

## Convex

```ts
import { ConvexReactClient } from "convex/react";
import { wireAveSessionToConvex } from "@ave-id/sdk/convex";

const convex = new ConvexReactClient(process.env.NEXT_PUBLIC_CONVEX_URL!);

await session.hydrate();
wireAveSessionToConvex(convex, session);
```

Call `session.setTokensFromResponse` after login. `wireAveSessionToConvex` passes a function that calls `getValidIdToken()` so Convex always receives a valid token.

## Expo

Use **`createSecureStoreAdapter`** with `expo-secure-store`, and optionally **`onExpoAppForegroundRefresh`** from `@ave-id/sdk/expo-lifecycle` with `AppState` to nudge refresh when returning from background.

```ts
import * as SecureStore from "expo-secure-store";
import { AveSession, createSecureStoreAdapter } from "@ave-id/sdk";
import { onExpoAppForegroundRefresh } from "@ave-id/sdk/expo-lifecycle";
import AppState from "react-native";

const session = new AveSession({
  oauth: { clientId: "...", redirectUri: "myapp://callback" },
  storage: createSecureStoreAdapter(SecureStore),
});

onExpoAppForegroundRefresh(AppState, session);
```

## BFF (HttpOnly refresh)

Pass **`customRefresh`** in `AveSession` options to POST to your own `/api/session/refresh` that reads an HttpOnly cookie and returns a **`TokenResponse`-shaped** JSON body. The client never stores the refresh token.

## E2EE

Session management does not replace E2EE setup. After OAuth, handle **`#app_key`** and related rules from [End-to-end encryption](/guides/end-to-end-encryption) and the [E2EE integration checklist](/guides/e2ee-integration-checklist).
