## API host

- Base URL: `https://api.aveid.net`

<Note>
	Use `api.aveid.net` for API calls and `aveid.net` for user-facing auth pages.
</Note>

## Request conventions

- JSON request bodies
- OAuth token endpoint expects **camelCase** body fields
- Bearer auth where applicable

<CodeGroup>
```bash OAuth token request
curl -X POST https://api.aveid.net/api/oauth/token \
	-H "Content-Type: application/json" \
	-d '{
		"grantType": "authorization_code",
		"code": "AUTHORIZATION_CODE",
		"redirectUri": "https://yourapp.com/callback",
		"clientId": "YOUR_CLIENT_ID",
		"codeVerifier": "PKCE_VERIFIER"
	}'
```

```bash Userinfo request
curl https://api.aveid.net/api/oauth/userinfo \
	-H "Authorization: Bearer ACCESS_TOKEN"
```
</CodeGroup>

## Auth models by endpoint family

### Public OAuth/OIDC

- Discovery and JWKS: public
- `/api/oauth/token`: public (client credentials / PKCE validated in payload)
- `/api/oauth/userinfo`: requires bearer access token
- `/api/oauth/app/:clientId`, `/api/oauth/resource/:resourceKey`: public metadata

### Session-authenticated OAuth management

Requires authenticated Ave session (bearer session token or Ave session cookie):

- `/api/oauth/authorize`
- `/api/oauth/authorizations`
- `/api/oauth/authorization/:clientId`
- `/api/oauth/delegations`

### Signing

- Public app endpoints: create request, poll status, verify, public key lookup
- Session-authenticated identity-management endpoints: key management + end-user sign/deny request routes

<Tabs>

## OIDC endpoints

- `GET /.well-known/openid-configuration`
- `GET /.well-known/jwks.json`
- `GET /.well-known/webfinger?resource=...`

## CORS/origin behavior summary

OAuth and OIDC paths are CORS-enabled with controlled origin resolution. Session-protected mutating requests with cookies enforce origin allowlisting.

<Warning>
	If you rely on cookie-based session auth for mutating endpoints, requests from disallowed origins are rejected.
</Warning>
