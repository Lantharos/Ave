---
title: "App keys (shared secret) endpoints"
description: "Identity encryption key lookup, shared-secret transfer contracts, recipient claim, and access retrieval. Same feature as App keys in the Ave dashboard (dashboard is read-only; apps create transfers via these routes)."
---

Base URL:

```text
https://api.aveid.net
```

## Public endpoints

### `GET /api/encryption/public-key/:handle`

Look up an identity's public encryption key by handle.

```bash
curl https://api.aveid.net/api/encryption/public-key/alice
```

Response:

```json
{
  "identityId": "11111111-1111-1111-1111-111111111111",
  "handle": "alice",
  "publicKey": "BASE64_SPKI_PUBLIC_KEY",
  "createdAt": "2026-03-30T20:00:00.000Z"
}
```

### `GET /api/shared-secrets/transfers/:claimToken`

Public. Resolve claim-link metadata before login or claim (used by the hosted claim page and by apps showing an invite preview).

Returns:

- transfer status
- target handle
- descriptor metadata
- owner identity metadata

Hosted claim URLs are built by the API as `{WEB_ORIGIN}/shared/claim?token=...` (e.g. `https://aveid.net/shared/claim?token=...` in production).

## Session-authenticated endpoints

All endpoints below require an Ave session token or session cookie.

### `GET /api/encryption/keys/:identityId`

Return the authenticated user's stored encryption key envelope for an identity.

Response includes:

- `wrapMode`: `master` or `oauth_app_key` (legacy rows behave as `master`).
- `oauthAppId` when the primary row was registered as oauth-primary.
- `oauthWraps`: extra ciphertext rows `{ oauthAppId, deviceKey, encryptedPrivateKey, createdAt }` when a master-wrapped primary also has third-party wraps.

### `POST /api/encryption/keys/:identityId`

Store a new identity encryption keypair generated client-side.

Default (`wrap_mode` omitted): **master**-style envelope (Ave web). Returns **409** if a row already exists.

**Third-party / OAuth wrap** ŌĆö set `wrap_mode` to `oauth_app_key`. The server validates **base64** and **size bounds** only; it never decrypts `encryptedPrivateKey`.

```json
{
  "publicKey": "BASE64_SPKI_PUBLIC_KEY",
  "encryptedPrivateKey": "YOUR_OPAQUE_AES_GCM_BLOB",
  "wrap_mode": "oauth_app_key",
  "oauth_app_id": "OAUTH_APP_UUID",
  "device_key": "optional-slot-id"
}
```

If the identity **already** has a **master**-wrapped primary key, this call **adds or updates** an OAuth wrap when `publicKey` matches the existing SPKI (see [E2EE OAuth apps & SPAs](/guides/e2ee-oauth-shared-secrets-spas)).

### `PUT /api/encryption/keys/:identityId`

Rotate or upsert the encryption keypair for an identity. Accepts the same optional `wrap_mode`, `oauth_app_id`, and `device_key` fields as `POST` for OAuth-aware updates.

## Shared secret lifecycle

### `POST /api/shared-secrets`

Create a logical shared secret record and the founder's own encrypted copy.

```json
{
  "identityId": "OWNER_IDENTITY_UUID",
  "kind": "app_scoped",
  "appId": "APP_UUID",
  "resourceKey": "vault.root",
  "label": "Citadel Vault Root Key",
  "encryptedSecret": "AES_GCM_WRAPPED_FOR_OWNER"
}
```

### `GET /api/shared-secrets`

List:

- created shared secrets for the caller's identities
- received recipient access envelopes already stored in Ave

### `POST /api/shared-secrets/:id/transfers`

Create a transfer contract for an existing Ave handle. The caller must be either the **secret owner** identity or an identity that has **already claimed** recipient access for this secret (so teammates can invite others without the founder online).

```json
{
  "identityId": "OWNER_IDENTITY_UUID",
  "targetHandle": "alice",
  "encryptedSecretForTarget": "RECIPIENT_CIPHERTEXT",
  "senderPublicKey": "EPHEMERAL_SENDER_PUBLIC_KEY",
  "expiresInHours": 72,
  "returnUrl": "https://citadel.app/invite/accepted"
}
```

Response includes:

- transfer metadata
- `claimToken`
- `claimUrl`

If `returnUrl` is set on the transfer, Ave redirects there after a successful claim and appends claim result query params so the destination app can treat the claim as an invite acceptance callback.

### `POST /api/shared-secrets/transfers/:claimToken/claim`

Validate the currently authenticated identity against the transfer target and return the encrypted recipient payload.

```json
{
  "identityId": "RECIPIENT_IDENTITY_UUID"
}
```

### `POST /api/shared-secrets/:id/finalize-recipient-storage`

Persist the recipient-owned wrapped copy after local decryption and re-wrap. The ciphertext is **opaque** to Ave; it must match whatever your client uses when reading `GET /api/shared-secrets/access` (Ave master wrap, app-key HKDF wrap, etc.).

```json
{
  "identityId": "RECIPIENT_IDENTITY_UUID",
  "transferId": "TRANSFER_UUID",
  "encryptedSecretForRecipient": "AES_GCM_WRAPPED_FOR_RECIPIENT"
}
```

Hosted `/shared/claim` supports `finalize_wrap=citadel_v1`, `resource_key`, and embed-only `defer_finalize=1` ŌĆö see [E2EE OAuth apps & SPAs](/guides/e2ee-oauth-shared-secrets-spas).

### `GET /api/shared-secrets/access`

Return recipient-owned envelopes for the caller's identities.

Optional query params:

- `kind=app_scoped|global`
- `appId=...`
- `resourceKey=...`

Example:

```bash
curl "https://api.aveid.net/api/shared-secrets/access?kind=app_scoped&appId=APP_UUID&resourceKey=vault.root" \
  -H "Authorization: Bearer SESSION_TOKEN"
```

## v1 rules

- Transfers are created **from your app** with the userŌĆÖs session; the dashboard does not create them.
- Target handle must already exist in Ave.
- Target identity must already have an **identity encryption** key (`GET /api/encryption/public-key/:handle` must succeed), or transfer creation will fail when encrypting for the recipient.
- Claiming identityŌĆÖs handle must match the transfer target exactly.
- Claim tokens expire; treat claim URLs as bearer secrets.
- Recipient storage is per **identity id**, not per handle string.

<Warning>
Treat the claim URL like a bearer secret. Anyone who can open it and successfully authenticate as the targeted handle can claim the transfer.
</Warning>
