In the Ave dashboard this feature appears as **App keys**. APIs and the SDK use the name **shared secrets** (`/api/shared-secrets`). **Creating transfers always happens from your app** (using the session token you get after the user signs in)—not from the dashboard, which only shows a read-only summary.

Use these helpers when your app needs to:

- look up another identity’s encryption **public** key by handle
- register a shared secret and start a transfer to another Ave handle
- claim a transfer and store the recipient’s wrapped copy in Ave
- list secrets you created or envelopes you received, optionally filtered by app / `resourceKey`

```bash
npm install @ave-id/sdk
```

## Prerequisites

- **Session token** — user must be signed in; pass the same bearer token your app uses for other Ave APIs.
- **Identity encryption keypair** — both the sender and the target identity need an encryption key stored in Ave before a transfer can succeed. If `GET /api/encryption/public-key/:handle` returns 404 for the target, they must sign in on Ave (or use your app’s flow) so their client can create `POST /api/encryption/keys/:identityId` once.
- **`app_scoped` secrets** — set `kind: "app_scoped"` and `appId` to your OAuth app’s UUID from the developer portal. Use `resourceKey` when your app has multiple stable slots (e.g. vaults).

## Identity encryption lookup

### `getIdentityEncryptionPublicKey(config, handle)`

Exported from `@ave-id/sdk` (main entry).

Returns the public encryption key for an Ave handle.

```ts
import { getIdentityEncryptionPublicKey } from "@ave-id/sdk";

const recipient = await getIdentityEncryptionPublicKey(
  { issuer: "https://aveid.net" },
  "alice"
);

console.log(recipient.identityId);
console.log(recipient.publicKey);
```

```ts
interface IdentityEncryptionPublicKey {
  identityId: string;
  handle: string;
  publicKey: string;
  createdAt: string;
}
```

## Listing (same data as the dashboard)

### `listSharedSecrets(config, sessionToken)`

Returns `created` (secrets owned by the user’s identities, with transfer rows) and `received` (recipient envelopes).

```ts
import { listSharedSecrets } from "@ave-id/sdk";

const result = await listSharedSecrets(
  { issuer: "https://aveid.net" },
  sessionToken
);

for (const secret of result.created) {
  console.log(secret.label, secret.transfers.length);
}
```

## Founder flow (high level)

### `createSharedSecretWithTransfer(params)`

Exported from **`@ave-id/sdk/client`**. Creates the shared secret row and the transfer in one call.

You must supply **`encryptedSecretForOwner`** — the ciphertext Ave stores for the owner’s identity (your app wraps the plaintext with the owner’s master key / your chosen format). The SDK also takes **`secret`** (plaintext) and encrypts it locally for the recipient before calling the transfer API.

```ts
import { createSharedSecretWithTransfer } from "@ave-id/sdk/client";

const result = await createSharedSecretWithTransfer({
  issuer: "https://aveid.net",
  sessionToken,
  ownerIdentityId,
  targetHandle: "alice",
  kind: "app_scoped",
  appId,
  resourceKey: "vault.root",
  label: "Team vault",
  returnUrl: "https://yourapp.com/invite/accepted",
  encryptedSecretForOwner,
  secret: vaultRootPlaintext,
});
```

Returns secret metadata, transfer metadata, **`claimToken`**, and **`claimUrl`**.

### Claim URL shape

The server builds:

```text
{WEB_ORIGIN}/shared/claim?token={claimToken}
```

Production uses the Ave web origin (e.g. `https://aveid.net`). Open the link in a browser; the recipient signs in and completes claim (your app can also drive claim programmatically—see below).

If **`returnUrl`** is set, after a successful claim Ave redirects there and appends:

- `shared_secret_claimed=1`
- `shared_secret_id=...`
- `transfer_id=...`
- `identity_id=...`

### Lower-level founder helpers (`@ave-id/sdk` / `@ave-id/sdk/client`)

- From **`@ave-id/sdk`**: `createSharedSecret`, `createSharedSecretTransfer`
- From **`@ave-id/sdk/client`**: `createSharedSecretPackage`, `encryptSharedSecretForIdentity`

### Teammate invites another teammate

`createSharedSecretWithTransfer` always **creates a new** shared secret row (founder flow). If someone has **already claimed** access and wants to invite the next person to the **same** secret, decrypt their envelope locally, then call `createSharedSecretPackage` + `createSharedSecretTransfer` with the **same** `sharedSecretId` and their `identityId` as the sender. The server allows transfers from the owner or from any identity with active recipient access.

## Recipient flow

### `claimAndStoreSharedSecret(params)`

Exported from **`@ave-id/sdk/client`**. Claims the transfer, decrypts with the recipient’s identity private key, re-wraps via your callback, then calls finalize.

```ts
import {
  claimAndStoreSharedSecret,
  importIdentityEncryptionPrivateKey,
} from "@ave-id/sdk/client";

const recipientPrivateKey = await importIdentityEncryptionPrivateKey(
  recipientPrivateKeyPkcs8Base64
);

await claimAndStoreSharedSecret({
  issuer: "https://aveid.net",
  sessionToken,
  claimToken,
  recipientIdentityId,
  recipientPrivateKey,
  wrapForRecipientStorage: encryptForYourStorage,
});
```

### Lower-level recipient helpers

- **`@ave-id/sdk`**: `resolveSharedSecretTransfer` (public), `claimSharedSecretTransfer`, `finalizeSharedSecretRecipientStorage`
- **`@ave-id/sdk/client`**: `decryptSharedSecretFromTransfer`

## Recipient access lookup

### `getSharedSecretAccess(config, sessionToken, params?)`

Exported from `@ave-id/sdk`. Returns recipient-owned envelopes, optionally filtered.

```ts
import { getSharedSecretAccess } from "@ave-id/sdk";

const access = await getSharedSecretAccess(
  { issuer: "https://aveid.net" },
  sessionToken,
  {
    kind: "app_scoped",
    appId: "YOUR_APP_ID",
    resourceKey: "vault.root",
  }
);
```

```ts
interface RecipientSharedSecretEnvelope {
  id: string;
  sharedSecretId: string;
  identityId: string;
  encryptedSecret: string;
  claimedAt?: string | null;
  descriptor: {
    id: string;
    kind: "app_scoped" | "global";
    appId?: string | null;
    resourceKey?: string | null;
    label?: string | null;
    appName?: string | null;
  };
  owner: {
    identityId: string;
    handle: string;
    displayName: string;
  };
}
```

## What the SDK does not do

The SDK runs the Ave API flow and transfer crypto. Your app still decides how to:

- encrypt **`encryptedSecretForOwner`** before `createSharedSecret` / `createSharedSecretWithTransfer`
- wrap plaintext after claim in **`wrapForRecipientStorage`**
