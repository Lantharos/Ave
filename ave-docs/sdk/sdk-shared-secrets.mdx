---
title: "@ave-id/sdk — Shared Secrets"
description: "Create, transfer, claim, and retrieve shared secrets through the Ave SDK."
---

Use the shared-secret SDK helpers when your app needs to:

- look up an identity encryption public key by handle
- create and transfer a shared secret from the founder side
- claim and finalize a transfer from the recipient side
- list the caller's shared secrets or recipient access
- filter recipient access for a specific app or `resourceKey`

```bash
npm install @ave-id/sdk
```

## Identity encryption lookup

### `getIdentityEncryptionPublicKey(config, handle)`

Returns the public encryption key for an Ave handle.

```ts
import { getIdentityEncryptionPublicKey } from "@ave-id/sdk";

const recipient = await getIdentityEncryptionPublicKey(
  { issuer: "https://aveid.net" },
  "alice"
);

console.log(recipient.identityId);
console.log(recipient.publicKey);
```

```ts
interface IdentityEncryptionPublicKey {
  identityId: string;
  handle: string;
  publicKey: string;
  createdAt: string;
}
```

## Shared secret listing

### `listSharedSecrets(config, sessionToken)`

Returns the caller's created shared secrets plus received access envelopes.

```ts
import { listSharedSecrets } from "@ave-id/sdk";

const result = await listSharedSecrets(
  { issuer: "https://aveid.net" },
  sessionToken
);

for (const secret of result.created) {
  console.log(secret.label, secret.transfers.length);
}
```

The response includes:

- `created`: secrets owned by one of the caller's identities
- `received`: recipient envelopes already stored in Ave for the caller's identities

## Founder flow helpers

### `createSharedSecretWithTransfer(params)`

Creates the shared secret record and transfer contract in one SDK call.

```ts
import { createSharedSecretWithTransfer } from "@ave-id/sdk/client";

const result = await createSharedSecretWithTransfer({
  issuer: "https://aveid.net",
  sessionToken,
  ownerIdentityId,
  targetHandle: "alice",
  kind: "app_scoped",
  appId,
  resourceKey: "vault.root",
  label: "Citadel Vault Root Key",
  returnUrl: "https://citadel.app/invite/accepted",
  encryptedSecretForOwner,
  secret: vaultRootPlaintext,
});
```

Returns:

- created shared secret metadata
- transfer metadata
- claim token
- claim URL

If `returnUrl` is provided, Ave redirects there after a successful claim and appends:

- `shared_secret_claimed=1`
- `shared_secret_id=...`
- `transfer_id=...`
- `identity_id=...`

### Lower-level founder helpers

- `createSharedSecret(...)`
- `createSharedSecretTransfer(...)`
- `createSharedSecretPackage(...)`
- `encryptSharedSecretForIdentity(...)`

## Recipient flow helpers

### `claimAndStoreSharedSecret(params)`

Claims a transfer, decrypts it locally, and finalizes recipient storage after calling your wrap callback.

```ts
import {
  claimAndStoreSharedSecret,
  importIdentityEncryptionPrivateKey,
} from "@ave-id/sdk/client";

const recipientPrivateKey = await importIdentityEncryptionPrivateKey(
  recipientPrivateKeyPkcs8Base64
);

await claimAndStoreSharedSecret({
  issuer: "https://aveid.net",
  sessionToken,
  claimToken,
  recipientIdentityId,
  recipientPrivateKey,
  wrapForRecipientStorage: encryptForYourStorage,
});
```

### Lower-level recipient helpers

- `resolveSharedSecretTransfer(...)`
- `claimSharedSecretTransfer(...)`
- `finalizeSharedSecretRecipientStorage(...)`
- `decryptSharedSecretFromTransfer(...)`

## Recipient access lookup

### `getSharedSecretAccess(config, sessionToken, params?)`

Returns recipient-owned shared secret envelopes, optionally filtered by secret type or app scope.

```ts
import { getSharedSecretAccess } from "@ave-id/sdk";

const access = await getSharedSecretAccess(
  { issuer: "https://aveid.net" },
  sessionToken,
  {
    kind: "app_scoped",
    appId: "YOUR_APP_ID",
    resourceKey: "vault.root",
  }
);
```

```ts
interface RecipientSharedSecretEnvelope {
  id: string;
  sharedSecretId: string;
  identityId: string;
  encryptedSecret: string;
  claimedAt?: string | null;
  descriptor: {
    id: string;
    kind: "app_scoped" | "global";
    appId?: string | null;
    resourceKey?: string | null;
    label?: string | null;
    appName?: string | null;
  };
  owner: {
    identityId: string;
    handle: string;
    displayName: string;
  };
}
```

## What the SDK does not do

The SDK handles the Ave-facing flow and transfer crypto, but your app still owns the storage wrapping strategy for:

- the founder's stored copy
- the recipient's stored copy after claim

## Recommended app flow

1. Encrypt the founder copy using your app's storage model.
2. Call `createSharedSecretWithTransfer`.
3. Send the claim URL to the recipient.
4. On the recipient side, call `claimAndStoreSharedSecret`.
5. Later, use `getSharedSecretAccess` to load the recipient envelope for the active identity.
